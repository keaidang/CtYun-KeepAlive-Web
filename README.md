# CtYun KeepAlive - 天翼云电脑与云手机多账号网页保活管理器

`CtYun KeepAlive` 是一款专门为**天翼云电脑（CtYun Desktop）**和**天翼云手机（CtYun Phone）**设计的多账号保活工具。该项目通过模拟客户端连接协议、维护底层 WebSocket 链路并动态响应保活质询，有效防止设备因长时间无操作被系统自动关闭或休眠。

本系统配备了现代化的**深色玻璃拟态（Glassmorphism）Web 控制面板**，支持实时日志滚动、动态账号管理以及设备首次绑定的网页短信交互。

---

## ✨ 核心特性

- 📱 **云手机与云电脑双支持**：根据天翼云接口返回的 `OsType` 自动判断设备类型。对云手机自动适配 `pm.ctyun.cn` 接口进行 WebSocket 握手，确保连接稳定。
- 👥 **多账号高并发保活**：独立管理每个账号下的所有设备，使用 `Task` 异步并发运行，账号互不干扰。
- 🎨 **玻璃拟态 Web 控制面板**：基于现代前端设计美学，集成指标汇总卡片、账号状态卡片和操作按钮。
- ⚙️ **图形化账号管理**：支持在网页端直接添加、启动、停止或删除保活账号，无需繁琐地手动修改后台配置文件。
- 🔒 **安全登录机制**：具备登录验证拦截。默认后台管理密码为 `admin`，支持在网页端直接修改密码并实时持久化。
- 📟 **极客风实时日志终端**：前端通过 SSE (Server-Sent Events) 管道实时同步后端保活日志，支持核心包过滤上色，排查故障一目了然。
- ⛓️ **一键直达官方客户端**：提供快速跳转至天翼云电脑官方 Web 端（`https://pc.ctyun.cn/`）的发光按钮。
- 🐳 **Docker 容器化与独立发布**：支持通过 Linux 独立二进制文件双击无依赖运行，也提供 Docker 镜像打包，支持挂载持久化目录。

## 🚀 部署与运行

### 1. 使用 Docker 部署 (推荐)

项目已提供配置好的 `Dockerfile`，将使用 `dotnet/aspnet:8.0` 作为运行环境，并默认在 `8080` 端口开启 Web 服务器。

**拉取与运行容器**：
```bash
# 克隆项目并进入目录
git clone https://github.com/your-username/CtYun-KeepAlive.git
cd CtYun-KeepAlive

# 构建 Docker 镜像
docker build -t ctyun-keepalive ./CtYun

# 运行 Docker 容器并挂载数据文件夹以实现设备码持久化
docker run -d \
  --name ctyun \
  -p 8080:8080 \
  -v "$(pwd)/ctyun-data:/app/data" \
  --restart always \
  ctyun-keepalive
```
*启动后，在浏览器中访问 `http://<服务器IP>:8080` 即可访问控制台。*

---

### 2. Linux 独立二进制文件运行 (无依赖)

若您的 Linux 服务器上未安装 Docker 或 .NET 运行环境，可直接通过发布包解压运行：

```bash
# 1. 解压包
unzip CtYun-linux-x64.zip -d ctyun
cd ctyun

# 2. 授予可执行权限
chmod +x CtYun

# 3. 自定义 Web 端口并常驻后台运行 (例如使用 9000 端口)
PORT=9000 nohup ./CtYun > ctyun.log 2>&1 &
```
*提示：解压后，请确保 `CtYun` 可执行程序和 `wwwroot` 静态文件夹处于同级目录下。*

## 🔧 配置说明

程序运行时会在数据目录中自动读取或创建 `accounts.json`。通过 Web 控制面板修改账号密码或保活配置后，后端将自动同步更新该文件。

### `accounts.json` 结构示例：
```json
{
  "keepAliveSeconds": 60,
  "adminPassword": "你的管理密码",
  "accounts": [
    {
      "name": "我的账号1",
      "user": "189xxxxxxxx",
      "password": "你的天翼云登录密码1",
      "deviceCode": "web_自行生成的32位随机字符"
    }
  ]
}
```

- **`keepAliveSeconds`**：保活强制重连周期，默认 `60` 秒。
- **`adminPassword`**：后台网页登录验证密码（默认为 `admin`）。
- **`deviceCode`**：设备识别码。当您在 Web 控制台添加账号时，留空系统会自动为您的账号随机生成并保存至本地 `devices/` 文件夹中。为了防止每次容器重建后重新绑定设备，请务必保证宿主机挂载目录的持久化。

## 🔒 设备首次绑定说明 (短信验证码)

当天翼云检测到账号在“新设备”上登录时，会触发短信验证码流程：
1. 在网页端点击“添加保活账号”并输入账号密码。
2. 页面会平滑过渡并弹出**验证码输入框**，同时官方会发送短信验证码到您的手机。
3. 在网页中输入短信验证码并点击确认，系统将调用 `binding` 接口自动向天翼云网关完成当前随机设备码的绑定授权。
4. 绑定成功后，账号保活协程将立即并发启动，后续自动登录时不再需要短信验证码。

## 🛠️ 技术内幕与保活原理

1. **反爬虫鉴权签名 (ctg-signaturestr)**：
   每次客户端 API 请求时，网关会校验其签名。我们通过公式 `${deviceType}${timestamp}${tenantId}${timestamp}${userId}${version}${secretKey}` 将参数拼接并进行 MD5 校验，自动在请求 Header 中附带完整的 `ctg-` 参数链，以确保安全穿透云端认证网关。
2. **WebSocket 二进制心跳响应**：
   项目连入连接网关后，系统通过自定义的 `Type + Length + Body` 二进制打包协议进行低级握手。
3. **RSA-OAEP 握手质询**：
   当网关下发带有 `"REDQ"` 特征的保活数据时，客户端自动拦截并提取服务端的 RSA 公钥模数 $N$ 及指数 $E$，在本地实时使用 RSA-OAEP $MGF1$ 掩码填充生成 20 字节随机种子的密文并发回，以此维持天翼云网关的保活检验。
